Lossless Coding & Entropy

Ch 1.1 + 1.4 · Interactive Notes Originals ↗

We use data compression thousands of times a day — every text, image, and video is compressed before it travels between devices (GZIP, JPEG, PNG, H264…). These schemes look domain-specific, but share deep, common ideas. Here we build the first one from scratch — and discover the hard limit on how far compression can go.

Part I

From fixed to variable-length codes

Start with a deliberately simple scenario.

Every character of a large text file is drawn independently and uniformly from $S=\{A,B,C,D\}$, i.e. $P(A)=P(B)=P(C)=P(D)=\tfrac14$. Say the file has 1,000,000 letters.

Stored as ASCII, each symbol takes 8 bits, so the file is 1,000,000 bytes. But the four ASCII codes all begin with the same 1000… — redundant when we only have four symbols. The question: can we make the file smaller, losslessly? We restrict ourselves to symbol codes — a fixed binary codeword per symbol.

Fixed-bitwidth encoding

With only $k$ distinct symbols we need just $\lceil \log_2 k\rceil$ bits each. Toggle which symbols actually appear:

So a uniform 4-symbol file compresses 4× for free. But this only helps when symbols are completely absent. What if every symbol appears, just in very different proportions?

Quiz — Beyond fixed bitwidth With $P(A)=P(B)=0.49$ and $P(C)=P(D)=0.01$, can a symbol code beat 2 bits/symbol?

Fixed-width can't: it always spends $\lceil\log_2 4\rceil=2$ bits regardless of probabilities. But we mostly see A and B, so we'd like to approach 1 bit/symbol. The fix: variable-length codes — short codewords for frequent symbols, longer for rare ones.

Why is this even decodable?

Fixed-width decoding was trivial: chop every 2 bits. With variable lengths we don't know where one codeword ends. Encode a string, then decode it bit-by-bit:

The decoder reads bits until the buffer matches a codeword, emits it, and resets. It works because no codeword is a prefix of another — the moment we match, we can commit with zero ambiguity.

Key idea — Prefix-free codes. If no codeword is a prefix of any other, the symbol code is prefix-free (a.k.a. instantaneous) and is always losslessly and instantaneously decodable.

Two facts let us reason about all prefix-free codes at once. First, a prefix-free code is a binary tree with symbols at the leaves, so the codeword length $\ell_i$ is a leaf depth. Second, the Kraft inequality says a set of lengths is realizable by a prefix-free code if and only if

$\displaystyle \sum_{i=1}^{k} 2^{-\ell_i} \le 1.$

That turns "design a good code" into a clean optimization: choose lengths $\ell_i$ to minimize the average $\sum_i p_i \ell_i$ subject to Kraft. Which raises the real question — how small can $\mathbb{E}[L]$ possibly get?

Part II

Entropy: the limit of compression

To answer that, we take a short detour into information theory — the science of information, laid down by Claude Shannon in 1948. The central quantity is entropy.

Entropy. For a random variable $X$ over $\{1,\dots,k\}$ with distribution $P=\{p_1,\dots,p_k\}$, $$H(X)=\sum_{i=1}^{k} p_i\,\log_2\frac{1}{p_i}.$$

A second viewpoint is illuminating: define the negative log-likelihood $\mathrm{NLL}(X)=\log_2\frac{1}{P(X)}$. Then entropy is just its expectation, $$H(X)=\mathbb{E}_P\!\left[\log_2\frac{1}{P(X)}\right].$$ Read $\log_2\frac{1}{p_i}$ as the "surprise" of seeing symbol $i$: rare symbols are surprising (large value), common ones are not. Entropy is the average surprise per symbol.

Two key properties

Both follow from Jensen's inequality applied to the concave $\log_2$:

• Non-negativity: $H(X)\ge 0$, since each $\log_2\frac{1}{p_i}\ge0$.
• Upper bound: $H(X)\le \log_2 k$, with equality iff $X$ is uniform.

So entropy is largest exactly when the distribution is flat (maximal uncertainty) and drops toward $0$ as the distribution concentrates on one symbol (no surprise left).

The thumb rule, and why entropy is a lower bound

Plug the "probabilities" $q_i=2^{-\ell_i}$ from Kraft into the average length and apply the property $H(P)\le \mathbb{E}_P[\log_2\frac1{q(X)}]$. Everything collapses to one statement:

Theorem — Entropy lower-bounds average code length. Any prefix-free (indeed any uniquely-decodable) code with lengths $\ell_i$ satisfies $$\sum_{i=1}^{k} p_i\,\ell_i \;\ge\; H(X),$$ with equality iff $\;\ell_i=\log_2\frac{1}{p_i}$ for every $i$.

The equality condition is exactly the thumb rule $\ell_i \approx \log_2\frac{1}{p_i}$: give each symbol a codeword as long as its surprise. We can't usually hit it exactly (lengths must be integers), but rounding up gives the Shannon code, $\ell_i=\big\lceil\log_2\frac1{p_i}\big\rceil$, which is provably within one bit:

$\displaystyle H(X)\;\le\;\mathbb{E}[L_{\text{Shannon}}]\;\le\;H(X)+1.$

Notice the gap to entropy shrinks when probabilities are near powers of two, and the Shannon bar never crosses $H+1$.

Mismatched coding & KL-divergence. Design a code for the wrong distribution $Q$ and use it on $P$, and the average length becomes $H(P)+D_{\mathrm{KL}}(P\,\|\,Q)$, where $D_{\mathrm{KL}}(P\|Q)=\sum_i p_i\log_2\frac{p_i}{q_i}\ge 0$. The KL-divergence is precisely the penalty you pay for getting the model wrong — and it's zero only when $Q=P$.

Entropy shows up far beyond compression — as the limit on simulating a random variable from fair coins, on extracting fair coins from a biased one, and on biased binary search. In every case $H(X)$ is the "amount of information," in bits, that $X$ is worth.

Summary & next

Variable-length prefix-free codes beat fixed-width whenever the source is non-uniform, and the entropy $H(X)$ is the hard floor on average code length — reachable to within one bit by the Shannon code. What we still don't have is the optimal integer-length code. That's Huffman coding →
