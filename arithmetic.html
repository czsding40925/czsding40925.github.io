Arithmetic Coding

Interactive

Breaking the whole-bit barrier.

Huffman is optimal, yet stuck giving each symbol a whole number of bits — so it can sit up to a bit above entropy. Arithmetic coding stops coding symbol-by-symbol and encodes the entire message as one number, letting symbols share fractional bits.

Step 1

The idea: shrink an interval

A symbol that "deserves" 1.4 bits costs 2 under Huffman, so there's up to a 1-bit-per-symbol tax. Arithmetic coding pays its rounding cost just once for the whole file. Here's how:

Start with the interval $[0,1)$. For each symbol, shrink the interval to the sub-slice that symbol owns — each symbol gets a slice as wide as its probability. After the last symbol you're left with a tiny interval; sending any number inside it identifies the exact message. Two facts make this optimal:

• more probable message ⟶ bigger final interval ⟶ fewer bits to pin down.
• the final interval's width equals $P(\text{message}) = \prod_i p(x_i)$, so the cost is $\approx \log_2\frac{1}{P(\text{message})}$ bits — the message's exact information content.

With a skewed, longer message, arithmetic coding beats Huffman: Huffman pays its rounding tax on every symbol, while arithmetic pays a fixed ~2-bit overhead once for the entire message. (On a very short message that fixed overhead can make it look worse — the readout is honest about this, and shows the per-symbol limit it converges to.)

Why this matters: arithmetic coding separates the model (the probabilities) from the coder. Improve your probability model — even one that adapts as it reads — and the same coder turns it straight into near-optimal bits. That's why it underpins modern image and video codecs (JPEG, H.264/H.265).

Step 3

What's next

Arithmetic coding hits the entropy floor but is relatively slow. ANS (Asymmetric Numeral Systems) gets the same compression at Huffman-like speeds, which is why it powers Zstandard and LZMA. And everything so far assumed symbols are independent — real data has patterns, which context modeling and LZ77 exploit. Those demos are coming next.
