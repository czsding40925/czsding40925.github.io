ANS: Asymmetric Numeral Systems

Interactive

One number that remembers everything.

Arithmetic coding reaches the entropy floor but is slow. ANS hits the same compression at Huffman-like speeds — which is why it powers Zstandard, LZMA, and modern media. The trick: fold the entire message into a single growing integer, the state.

Step 1

Numbers already are a code

Think about how a normal number stores digits. To append a digit $d$ (base 10) you compute $x' = 10x + d$; to read it back you take $d = x \bmod 10$ and $x = \lfloor x/10\rfloor$. A single integer losslessly stores a whole sequence of digits, newest on the outside.

That works great when all 10 digits are equally likely. But symbols in real data aren't equally likely — and spending the same "room" on a rare symbol as a common one wastes space. ANS makes the number system asymmetric: common symbols grow the state a little, rare symbols grow it a lot — by exactly $\log_2\frac1p$ bits each, just like the entropy thumb rule.

Step 2

The rANS update

Give each symbol $s$ an integer frequency $f_s$, with total $M=\sum_s f_s$ and cumulative offset $c_s$. Encoding one symbol transforms the state $x$:

$x' = \left\lfloor \dfrac{x}{f_s}\right\rfloor \cdot M \;+\; (x \bmod f_s) \;+\; c_s.$

A frequent symbol (big $f_s$) barely changes $x$; a rare one (small $f_s$) multiplies it by roughly $M/f_s = 1/p_s$. Decoding is the exact inverse and pops symbols off in reverse order (last in, first out) — read the slot $x\bmod M$, find whose range it lands in, undo the step.

The state's bit-length climbs by about $\log_2\frac1{p_s}$ for each symbol — fractional bits and all — so ANS lands right at the entropy floor, just like arithmetic coding. The difference is speed: each step is a couple of integer operations and a table lookup, with no fractional arithmetic.

Step 4

Why it took over

For decades the choice was stark: Huffman (fast, but wastes up to a bit per symbol) or arithmetic coding (optimal, but slow). ANS — published by Jarek Duda in 2009 — gives you both at once. Its table-driven variant tANS turns the whole thing into a finite-state machine of lookups, which is why zstd and many modern codecs replaced Huffman with it. It's the entropy-coding stage sitting underneath a huge amount of today's data.
